Overview & Technology Summary

Why JavaScript & CSS? I chose vanilla JavaScript and CSS because they are the foundational building blocks of every website. Before reaching for React or Vue, understanding what happens natively in the browser gives you real control over what you're building.

JavaScript is a high-level, interpreted programming language that enables dynamic behaviour in web pages — responding to clicks, updating content without a page reload, animating elements, and communicating with APIs. It runs directly in the browser with zero installation.

CSS (Cascading Style Sheets) controls presentation: layout, colour, spacing, typography and — crucially — animations. Together, JS + CSS form the minimum viable toolkit for a compelling web experience.

End Goal: Build an interactive card dashboard component that responds to user input, animates in on load, and demonstrates DOM manipulation — all with zero frameworks.

Quick Facts

JavaScript was created in 1995 by Brendan Eich and standardised as ECMAScript. CSS was proposed by Håkon Wium Lie in 1994. Both are maintained under W3C/ECMA standards.

Real-world use

Google Docs uses vanilla JS extensively for the editor. Twitter's first UI was plain JS + CSS before React was invented. GitHub's page interactions are largely CSS-driven.

Installation & Setup

01

Install VS Code

Download and install Visual Studio Code from code.visualstudio.com. It's free and available on all platforms.

02

Install the Live Server Extension

In VS Code, open Extensions (Ctrl+Shift+X), search Live Server by Ritwick Dey, and click Install. This lets you see changes instantly.

03

Create Your Project Folder

Create a new folder called js-css-toolkit and open it in VS Code via File → Open Folder.

# In your terminal (optional)
mkdir js-css-toolkit
cd js-css-toolkit
code .
04

Create Your Three Files

Inside the folder, create these files:

js-css-toolkit/
├── index.html     ← Your HTML structure
├── style.css      ← Your styles & animations
└── app.js         ← Your JavaScript logic
05

Link CSS & JS in HTML

In your index.html, add the following boilerplate inside <head> and before </body>:

<!-- Inside <head> -->
<link rel="stylesheet" href="style.css">

<!-- Before </body> -->
<script src="app.js"></script>
06

Launch Live Server

Right-click on index.html in VS Code and choose Open with Live Server. Your browser will open at http://127.0.0.1:5500 and refresh on every save.

Minimal Working Example

This example builds an interactive card component: cards fade in on page load, and a button adds a new card dynamically — all in under 50 lines of code.

style.css

/* Fade-in animation */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.card {
  background: white;
  border: 1px solid #ddd;
  padding: 1.2rem;
  margin-bottom: 1rem;
  animation: fadeIn 0.5s ease forwards;
  opacity: 0; /* starts hidden */
}

.card:nth-child(1) { animation-delay: 0.1s; }
.card:nth-child(2) { animation-delay: 0.25s; }
.card:nth-child(3) { animation-delay: 0.4s; }

app.js

// Select the container & button
const container = document.querySelector('#card-list');
const btn = document.querySelector('#add-btn');
let count = 3;

// Add a new card on button click
btn.addEventListener('click', () => {
  count++;
  const card = document.createElement('div');
  card.className = 'card';
  card.textContent = `Card #${count} — dynamically added`;
  container.appendChild(card);

  // Reset animation so new card fades in
  card.style.animation = 'none';
  card.offsetHeight; // trigger reflow
  card.style.animation = '';
});

Common Errors & Fixes

ERROR: Cannot read properties of null (reading 'addEventListener')

This happens when the <script> tag is in <head> and runs before the HTML has loaded, so document.querySelector returns null.

✓ Fix

Move the <script> tag to just before </body>, or wrap your code in document.addEventListener('DOMContentLoaded', () => { ... }).

CSS animation plays once then element disappears

The animation ends but the element reverts to its initial state (opacity: 0) because animation-fill-mode is not set.

✓ Fix

Add animation-fill-mode: forwards (or use the shorthand animation: fadeIn 0.5s ease forwards). This keeps the final keyframe state.

Dynamically added card does not animate

Resetting the animation class alone does not re-trigger CSS animations because the browser batches style changes.

✓ Fix

Force a reflow between removing and re-adding the animation: card.style.animation = 'none'; card.offsetHeight; card.style.animation = ''; — reading offsetHeight forces the browser to flush styles.

Live Server not reflecting changes

Sometimes the browser cache serves the old file even after you save.

✓ Fix

Hard-refresh the browser with Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac). If that fails, disable browser cache in DevTools (Network tab → Disable cache, keep DevTools open).

Fonts not loading from Google Fonts

The <link> to Google Fonts must appear before your stylesheet, and you need an internet connection.

✓ Fix

Place the Google Fonts <link> as the first <link> in your <head>. For offline use, download the font files and host them locally.
